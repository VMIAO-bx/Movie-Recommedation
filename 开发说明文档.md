# 第一部分：实际开发过程

## 1. 项目背景与需求分析

### 1.1 项目背景

随着流媒体平台的蓬勃发展，影视资源数量呈爆炸式增长，用户面临严重的"信息过载"问题。如何从海量影视作品中高效地发现、筛选和管理自己感兴趣的内容，成为提升用户体验的核心课题。本系统旨在构建一个集电影浏览、搜索、收藏与观影标记于一体的 Web 应用平台，帮助用户便捷地管理个人观影生活。

### 1.2 功能性需求

系统功能需求划分为以下模块：

| 模块 | 功能点 | 描述 |
|------|--------|------|
| **电影浏览** | 热门电影展示 | 首页展示热门电影列表，支持响应式网格布局 |
| | 电影详情查看 | 点击电影卡片查看完整信息（简介、评分、上映日期等） |
| **电影搜索** | 关键词搜索 | 支持按电影名称实时搜索，提供自动补全建议 |
| **用户认证** | 注册 | 新用户创建账号（用户名、密码、昵称） |
| | 登录/登出 | 基于 JWT 的身份认证与登出 |
| | 会话持久化 | 页面刷新后自动恢复登录状态 |
| **个人收藏** | 添加收藏 | 对感兴趣的电影进行收藏 |
| | 取消收藏 | 移除已收藏的电影 |
| | 收藏列表 | 按时间倒序查看收藏的电影 |
| **观影标记** | 标记"想看" | 将电影加入待看列表 |
| | 标记"看过" | 标记已观看的电影 |
| | 状态切换 | 在"想看"与"看过"之间切换 |
| | 标记列表 | 按时间倒序查看标记列表 |
| **个人中心** | 个人信息管理 | 修改昵称、上传头像 |
| **界面体验** | 深色/浅色主题 | 支持一键切换深色/浅色模式，偏好持久化 |

### 1.3 非功能性需求

| 类别 | 要求 |
|------|------|
| **安全性** | 密码 BCrypt 加密存储；JWT 令牌认证；API 层统一鉴权拦截 |
| **可用性** | 前端 UI 加载骨架屏；网络异常可重试；Token 过期自动提示 |
| **兼容性** | 支持主流现代浏览器（Chrome、Firefox、Edge） |
| **可维护性** | 后端分层架构（Controller → Service → Mapper）；前端组件化开发 |
| **响应时间** | API 响应时间 < 3s；页面首次加载 < 5s |

## 3. 系统架构设计

### 3.1 总体架构

系统采用**前后端分离**的 B/S 架构，前端与后端通过 HTTP/JSON 进行数据交互，前端独立部署于 Vite 开发服务器（端口 5173），后端部署于 Spring Boot 内嵌 Tomcat（端口 8080）。电影数据来源于 TMDB（The Movie Database）开放 API，通过 Vite 代理转发解决跨域与网络访问问题。

```
┌─────────────────────────────────────────────────────────┐
│                      浏览器客户端                         │
│            http://localhost:5173                        │
└────────────┬──────────────────────────────┬─────────────┘
             │                              │
    前端资源请求                      API 数据请求
    (Vite Dev Server)                (Axios + JWT Token)
             │                              │
             ▼                              ▼
┌─────────────────────┐     ┌──────────────────────────────┐
│   Vue 3 前端应用     │     │   Spring Boot 后端服务         │
│   Port: 5173        │     │   Port: 8080                  │
│                     │     │                              │
│ ┌───────────────┐   │     │ ┌──────────────────────────┐ │
│ │ Vue Router    │   │     │ │ Interceptor (JWT 鉴权)    │ │
│ │ (SPA 路由)    │   │     │ ├──────────────────────────┤ │
│ ├───────────────┤   │     │ │ Controller (REST API)    │ │
│ │ Components    │   │     │ ├──────────────────────────┤ │
│ │ (Element Plus)│   │     │ │ Service (业务逻辑)       │ │
│ ├───────────────┤   │     │ ├──────────────────────────┤ │
│ │ Axios (HTTP)  │   │     │ │ Mapper (MyBatis Plus)    │ │
│ └───────────────┘   │     │ └──────────────────────────┘ │
└──────────┬──────────┘     └──────────────┬───────────────┘
           │                               │
           │ /tmdb 代理                    │ JDBC
           ▼                               ▼
┌─────────────────────┐     ┌──────────────────────────────┐
│ TMDB API (外部)      │     │ MySQL 数据库                  │
│ api.themoviedb.org  │     │ movie_recommendation         │
└─────────────────────┘     └──────────────────────────────┘
```

### 3.2 前端架构

前端采用 **Vue 3 单页应用（SPA）** 架构，遵循组件化开发模式：

- **视图层**（Views）：`Home.vue`、`Profile.vue` 两个顶层路由页面
- **组件层**（Components）：`NavBar`、`MovieCard`、`MovieDetail`、`LoginModal` 四个可复用组件
- **基础设施层**：Axios 实例（请求/响应拦截器）、Vue Router（路由守卫）
- **状态管理**：不引入独立状态库，通过 `provide/inject` + `localStorage` + 组件 Props/Events 实现轻量级状态共享

```
App.vue (根组件 — 状态宿主)
 ├── NavBar.vue          ← 导航栏 + 搜索 + 主题切换
 ├── <router-view>
 │   ├── Home.vue        ← 首页（电影浏览 / 搜索结果）
 │   │   ├── MovieCard.vue    ← 电影卡片（海报、评分、操作）
 │   │   └── MovieDetail.vue  ← 电影详情弹窗
 │   └── Profile.vue     ← 个人中心（收藏 / 标记 / 设置）
 └── LoginModal.vue      ← 登录/注册弹窗（条件渲染）
```

### 3.3 后端架构

后端采用经典的 **分层架构**（Layered Architecture），各层职责清晰：

```
┌─────────────────────────────────────────┐
│              Controller 层               │
│   接收 HTTP 请求，参数校验，调用 Service   │
│   返回 Result<T> 统一响应                 │
├─────────────────────────────────────────┤
│              Service 层                   │
│   业务逻辑处理，事务管理                   │
│   调用 Mapper 层进行数据访问               │
├─────────────────────────────────────────┤
│              Mapper 层                    │
│   继承 MyBatis Plus BaseMapper<T>        │
│   提供通用 CRUD + Lambda 条件查询          │
├─────────────────────────────────────────┤
│              Entity 层                    │
│   数据库表对应的实体类（Lombok @Data）      │
│   通过 @TableName 映射数据库表             │
└─────────────────────────────────────────┘

横向切面：
├── Interceptor（拦截器）：JWT 认证
├── Config（配置）：CORS、BCrypt、MyBatis Plus、WebMvc
├── Common（通用）：统一响应、全局异常处理
└── Util（工具）：JWT 工具、ThreadLocal 用户上下文
```

## 4. 数据库设计

### 4.1 概念模型（E-R 图）

系统包含三个核心实体：**用户（User）**、**收藏（Favorite）**、**观影标记（Watchlist）**。一个用户可以拥有多条收藏记录和多条观影标记记录，每条收藏/标记记录关联一部外部电影（通过 `movie_id` 标识）。

```
┌──────────┐         ┌──────────────┐
│   User   │ 1 ─── N │  Favorite    │
│          │         │  movie_id    │ ───→ TMDB Movie
│          │         │  movie_title │
│          │         │  movie_poster│
└──────────┘         └──────────────┘
     │
     │ 1
     │
     N
┌──────────────┐
│  Watchlist   │
│  movie_id    │ ───→ TMDB Movie
│  movie_title │
│  movie_poster│
│  status      │ (想看/看过)
└──────────────┘
```

### 4.2 物理模型

**数据库**: `movie_recommendation`，字符集 `utf8mb4`，排序规则 `utf8mb4_unicode_ci`。

#### 用户表（user）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | 用户唯一标识 |
| `username` | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| `password` | VARCHAR(255) | NOT NULL | 密码（BCrypt 密文） |
| `nickname` | VARCHAR(100) | DEFAULT NULL | 昵称 |
| `avatar` | VARCHAR(500) | DEFAULT NULL | 头像 URL |
| `create_time` | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 注册时间 |

> 索引：`PRIMARY KEY (id)`, `UNIQUE KEY uk_username (username)`

#### 收藏表（favorite）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | 收藏记录 ID |
| `user_id` | BIGINT | NOT NULL | 用户 ID |
| `movie_id` | VARCHAR(50) | NOT NULL | TMDB 电影 ID |
| `movie_title` | VARCHAR(200) | NOT NULL | 电影标题 |
| `movie_poster` | VARCHAR(500) | DEFAULT NULL | 海报 URL |
| `create_time` | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 收藏时间 |

> 索引：`PRIMARY KEY (id)`, `UNIQUE KEY uk_user_movie (user_id, movie_id)`, `KEY idx_user_id (user_id)`  
> 约束：同一用户不可重复收藏同一电影

#### 观影标记表（watchlist）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | 标记记录 ID |
| `user_id` | BIGINT | NOT NULL | 用户 ID |
| `movie_id` | VARCHAR(50) | NOT NULL | TMDB 电影 ID |
| `movie_title` | VARCHAR(200) | NOT NULL | 电影标题 |
| `movie_poster` | VARCHAR(500) | DEFAULT NULL | 海报 URL |
| `status` | VARCHAR(20) | NOT NULL, DEFAULT '想看' | 状态（"想看"/"看过"） |
| `create_time` | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |

> 索引：`PRIMARY KEY (id)`, `UNIQUE KEY uk_user_movie_status (user_id, movie_id)`, `KEY idx_user_id (user_id)`, `KEY idx_status (status)`  
> 约束：同一用户对同一电影只能有一条标记记录，通过更新状态切换"想看"/"看过"

## 5. 接口设计

### 5.1 API 设计规范

- **协议**: HTTP
- **数据格式**: JSON
- **字符编码**: UTF-8
- **统一响应格式**:

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

**状态码体系**:

| code | 含义 |
|------|------|
| 200 | 操作成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 / Token 无效 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 5.2 接口清单

#### 系统接口

| 方法 | 路径 | 认证 | 说明 |
|------|------|:----:|------|
| GET | `/` | — | 健康检查，返回 `{app, version, status: "running"}` |
| ANY | `/error` | — | 全局错误处理，返回 404 JSON |

#### 用户模块

| 方法 | 路径 | 认证 | 请求体 | 响应 data | 说明 |
|------|------|:----:|--------|-----------|------|
| POST | `/api/user/register` | — | `{username, password, nickname}` | 用户信息对象 | 注册新用户 |
| POST | `/api/user/login` | — | `{username, password}` | `{token, user}` | 登录获取令牌 |
| GET | `/api/user/info` | ✅ | — | 用户信息对象 | 获取当前用户信息 |
| PUT | `/api/user/info` | ✅ | `{nickname?, avatar?}` | 更新后的用户信息 | 修改个人信息 |

#### 收藏模块

| 方法 | 路径 | 认证 | 请求体 | 响应 data | 说明 |
|------|------|:----:|--------|-----------|------|
| POST | `/api/favorite/add` | ✅ | `{movieId, movieTitle, moviePoster}` | 收藏记录 | 添加收藏 |
| POST | `/api/favorite/remove` | ✅ | `{movieId}` | `null` | 取消收藏 |
| GET | `/api/favorite/list` | ✅ | — | 收藏记录数组 | 获取收藏列表（按时间倒序） |

#### 观影标记模块

| 方法 | 路径 | 认证 | 请求体 | 响应 data | 说明 |
|------|------|:----:|--------|-----------|------|
| POST | `/api/watch/add` | ✅ | `{movieId, movieTitle, moviePoster, status}` | 标记记录 | 添加标记（`status`: "想看"/"看过"） |
| PUT | `/api/watch/update` | ✅ | `{movieId, movieTitle, moviePoster, status}` | 更新后的记录 | 修改标记状态 |
| POST | `/api/watch/remove` | ✅ | `{movieId}` | `null` | 移除标记记录 |
| GET | `/api/watch/list` | ✅ | — | 标记记录数组 | 获取标记列表（按时间倒序） |

## 6. 前端开发实现

### 6.1 项目初始化与工程化配置

**项目创建**

使用 `npm create vite@latest` 脚手架创建 Vue 3 项目，选择 Vue 模板并启用 TypeScript 支持级别的基础配置。项目采用 ES Module（`"type": "module"`）模式。

**依赖安装**

```bash
npm install vue-router@4 element-plus @element-plus/icons-vue axios
```

**Vite 配置**（`vite.config.js`）

```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: {
    port: 5173,
    open: true,              // 启动后自动打开浏览器
    proxy: {
      '/tmdb': {             // TMDB API 代理
        target: 'https://api.themoviedb.org',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/tmdb/, ''),
        timeout: 30000
      }
    }
  }
})
```

**关键设计决策**：由于 TMDB API 在国内直接访问可能超时或不稳定，通过在 Vite 开发服务器中配置代理（`/tmdb` → `https://api.themoviedb.org`）解决跨域与网络问题，前端请求 TMDB 时使用路径 `/tmdb/3/...` 即可。

### 6.2 路由设计

系统采用 Vue Router 的 HTML5 History 模式（`createWebHistory()`），共定义两个路由：

| 路径 | 组件 | 认证要求 | 说明 |
|------|------|:--------:|------|
| `/` | `Home.vue` | 公开 | 首页，电影浏览与搜索 |
| `/profile` | `Profile.vue` | 需登录 | 个人中心，收藏与标记管理 |

路由采用**懒加载**策略（动态 `import()`），实现代码分割，减小首屏加载体积：

```javascript
const routes = [
  {
    path: '/',
    name: 'Home',
    component: () => import('../views/Home.vue')
  },
  {
    path: '/profile',
    name: 'Profile',
    component: () => import('../views/Profile.vue'),
    meta: { requiresAuth: true }    // 路由元信息标记需认证
  }
]
```

路由守卫逻辑在 `App.vue` 中通过监听路由变化实现：当目标路由 `meta.requiresAuth` 为 `true` 且用户未登录时，弹出登录模态框而非强制跳转。

### 6.4 状态管理与数据流

系统**未引入** Pinia 或 Vuex 等独立状态管理库，基于 Vue 3 内置 API 实现轻量级状态管理：

```
状态宿主：App.vue
   ├── user: ref(null)           ← 当前用户信息
   ├── showLogin: ref(false)     ← 登录弹窗显示状态
   ├── searchKeyword: ref('')    ← 搜索关键词
   │
   ├── provide('user', user)     ← 向下注入
   ├── provide('showLogin', fn)  ← 向下注入
   │
   └── 持久化：localStorage
         ├── 'token'  ← JWT 令牌
         └── 'user'   ← 用户信息 JSON
```

**数据流路径**:

1. **登录**: `LoginModal` → 调用 API → `App.handleLoginSuccess()` → 更新 `user` ref + 写入 localStorage
2. **登出**: `NavBar` → `App.handleLogout()` → 清除 ref + 清除 localStorage
3. **搜索**: `NavBar` 输入 → emit `search` → `App.handleSearch()` → 更新 `searchKeyword` → Props 传给 `Home.vue`
4. **会话恢复**: `App.onMounted()` → 读取 localStorage token → 调用 `/api/user/info` 验证有效性 → 恢复或清除状态

### 6.5 网络请求层设计

在 `src/api/index.js` 中创建 Axios 实例，配置如下：

- **baseURL**: `http://localhost:8080/api`
- **timeout**: 10000ms
- **请求拦截器**：自动从 localStorage 读取 token 并附加 `Authorization: Bearer <token>` 请求头
- **响应拦截器**：
  - 业务错误（`code !== 200`）：显示错误提示；401/403 时清除本地 token 与用户信息
  - HTTP 网络错误：区分处理 401/403（静默清除）与网络中断（提示用户）；忽略已取消的请求错误（`ERR_CANCELED`）

该封装使得业务组件无需关心 Token 管理细节，只需关注 `request.get/post/put` 返回的 `data` 字段。

### 7.2 分层架构实现

**Controller 层**：负责接收 HTTP 请求，通过 `@RequestBody` + `@Valid` 进行参数校验，从 `CurrentUserHolder` 获取当前登录用户 ID，调用 Service 方法后将结果包装为 `Result<T>` 统一响应。

**Service 层**：采用**接口-实现分离**模式（`UserService` → `UserServiceImpl`），便于后续扩展与测试。业务逻辑使用 MyBatis Plus 的 `LambdaQueryWrapper` 进行类型安全的条件查询：

```java
// 示例：查询用户收藏列表
LambdaQueryWrapper<Favorite> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(Favorite::getUserId, userId)
       .orderByDesc(Favorite::getCreateTime);
List<Favorite> list = favoriteMapper.selectList(wrapper);
```

**Mapper 层**：继承 `BaseMapper<T>` 即可获得 `selectById`、`selectList`、`insert`、`delete`、`updateById` 等通用 CRUD 方法，无需编写 XML 映射文件，大幅减少代码量。

**DTO 层**：使用 `javax.validation` 注解（`@NotBlank`、`@Size`）在 Controller 层自动校验请求参数，非法请求直接返回 400 错误，不进入业务逻辑。

### 7.3 身份认证与授权

系统采用**无状态 JWT 认证**方案：

**认证流程**：

```
用户登录 → POST /api/user/login
         → 后端验证用户名密码（BCrypt.checkpw）
         → 生成 JWT Token（携带 userId + username，HS512 签名，7 天有效期）
         → 返回 {token, user}

后续请求 → Axios 拦截器附加 Authorization: Bearer <token>
         → LoginInterceptor.preHandle()
              ├── OPTIONS 请求 → 放行（CORS 预检）
              ├── 非 HandlerMethod → 放行（404 / 静态资源）
              ├── Token 缺失 → 401 JSON
              ├── Token 无效/过期 → 401 JSON
              └── Token 有效 → userId 存入 CurrentUserHolder（ThreadLocal）
                              → 请求处理
                              → afterCompletion() 清理 ThreadLocal
```

**拦截器注册**（`WebMvcConfig.java`）：

- 拦截路径：`/api/**`
- 排除路径：`/api/user/register`、`/api/user/login`

**关键设计**：在 `LoginInterceptor.preHandle()` 中通过 `handler instanceof HandlerMethod` 判断是否为实际的 Controller 方法。对于不存在的路径（如 `/api/xyz`），Spring MVC 会将它们映射到默认的 ResourceHttpRequestHandler，此时 `handler` 不是 `HandlerMethod` 实例，拦截器直接放行，让 Spring 的默认错误处理机制返回 404 JSON 而非 401。这解决了"不存在接口被拦截器误判为未认证"的问题。

**CurrentUserHolder** 使用 `ThreadLocal<Long>` 实现线程安全的用户上下文传递：

```java
public class CurrentUserHolder {
    private static final ThreadLocal<Long> USER_ID_HOLDER = new ThreadLocal<>();

    public static void setUserId(Long userId) { USER_ID_HOLDER.set(userId); }
    public static Long getUserId() { return USER_ID_HOLDER.get(); }
    public static void clear() { USER_ID_HOLDER.remove(); }
}
```

### 7.4 安全设计

| 安全措施 | 实现方式 |
|----------|----------|
| **密码加密** | `spring-security-crypto` 的 `BCryptPasswordEncoder`，注册时对明文密码进行哈希后存储，登录时使用 `encoder.matches(rawPassword, encodedPassword)` 验证 |
| **密码隐匿** | `User` 实体的 `password` 字段标注 `@JsonIgnore`，序列化 JSON 时永不输出 |
| **Token 安全** | JWT 采用 HS512 算法签名，密钥为项目专属随机字符串；设置 7 天过期时间防止长期有效 |
| **防重放** | Token 过期机制间接防护，未引入 Nonce 机制（不适用于无 Session 架构） |
| **CORS 安全** | `CorsFilter` 允许所有来源（开发阶段），生产环境应限制为前端域名 |

### 7.5 全局异常处理

`GlobalExceptionHandler` 类通过 `@RestControllerAdvice` 注解拦截全局异常：

- `MethodArgumentNotValidException` — `@Valid` 校验失败，返回 400 + 具体字段错误
- `Exception` — 兜底处理，记录日志并返回 500 通用错误

对于不存在的路径，利用 Spring Boot 的 `ErrorPageCustomizer` 机制，通过 `IndexController` 中映射 `/error` 端点返回自定义 404 JSON。

## 8. 开发过程中的问题与解决方案

| 序号 | 问题描述 | 原因分析 | 解决方案 |
|:----:|----------|----------|----------|
| 1 | 本地未安装 Maven，`mvn` 命令不可用 | Windows 系统默认不安装 Maven | PowerShell 下载 Maven 3.9.6 到临时目录，配置 `PATH` 环境变量 |
| 2 | `mvn` 运行时抛出 SSL 证书错误 | Java 8 默认 TLS 版本较低，不被 Maven Central 接受 | 添加 JVM 参数 `-Dhttps.protocols=TLSv1.2` |
| 3 | Maven 下载依赖速度极慢 | 默认仓库 `repo.maven.apache.org` 国内访问延迟高 | 在 `~/.m2/settings.xml` 中配置阿里云 Maven 镜像 |
| 4 | 编译时 `@NotBlank` 和 `@Valid` 注解报错找不到 | Spring Boot Web Starter 不包含 Validation Starter | 在 `pom.xml` 中添加 `spring-boot-starter-validation` 依赖 |
| 5 | 数据库 insert 后返回的 `createTime` 字段为 `null` | MyBatis Plus 默认不会自动填充时间字段 | 创建 `MyMetaObjectHandler` 实现 `MetaObjectHandler` 接口，在 `insertFill()` 中自动设置 `createTime = LocalDateTime.now()` |
| 6 | 根路径 `/` 返回 Spring Boot 白标错误页（Whitelabel Error Page） | Spring Boot 默认启用白标错误页，且无 `/` 的 Controller 映射 | 添加 `IndexController` 映射 `/` 返回 JSON 欢迎信息；在 `application.yml` 中设置 `error.whitelabel.enabled: false` |
| 7 | 不存在的 `/api/*` 路径返回 401 而非 404 | `LoginInterceptor` 拦截了所有 `/api/**` 请求，包括不存在的路径（它们被路由到默认 Handler），导致先触发认证失败 | 在 `LoginInterceptor.preHandle()` 中增加 `handler instanceof HandlerMethod` 判断，非 Controller 方法的请求直接放行，由 Spring 默认机制返回 404 |
| 8 | 前端 TMDB API 请求超时 | 国内网络访问 `api.themoviedb.org` 不稳定，受 GFW 干扰 | Vite 开发服务器配置代理 `/tmdb` → `https://api.themoviedb.org`，通过本地 Node.js 进程转发请求 |
| 9 | 页面刷新后用户登录状态丢失 | 登录信息仅存储在 Vue 响应式变量中，刷新即清空 | 将 Token 和用户信息持久化至 `localStorage`，`App.vue` 在 `onMounted` 中恢复并验证；Axios 拦截器自动读取 Token 附加到请求头 |

```yaml
    password: 你的MySQL密码        # 替换为实际密码
```

### 11.2 后端启动

**方式一：使用 Maven Wrapper（推荐，无需安装 Maven）**

```bash
cd movie-backend

# Windows
mvnw.cmd spring-boot:run -Dhttps.protocols=TLSv1.2

# Linux / macOS
./mvnw spring-boot:run -Dhttps.protocols=TLSv1.2
```

**方式二：使用系统 Maven**

```bash
cd movie-backend
mvn spring-boot:run -Dhttps.protocols=TLSv1.2
```

**方式三：在 IDE 中运行**

使用 IntelliJ IDEA 打开 `movie-backend` 目录，运行 `MovieBackendApplication` 类的 `main()` 方法。

**启动成功标志**：

控制台输出 `Tomcat started on port(s): 8080`，访问 `http://localhost:8080/` 返回：

```json
{
  "code": 200,
  "data": {
    "app": "Movie Recommendation Backend",
    "version": "1.0.0",
    "status": "running"
  }
}
```

**故障排查**：

| 症状 | 可能原因 | 解决方法 |
|------|----------|----------|
| 连接拒绝 `Connection refused` | MySQL 未启动 | 启动 MySQL 服务 |
| `Access denied for user` | 数据库密码错误 | 检查 `application.yml` 中密码 |
| `Unknown database` | 未执行 `init.sql` | 执行数据库初始化脚本 |
| `SSLHandshakeException` | JVM SSL 配置问题 | 确保添加 `-Dhttps.protocols=TLSv1.2` 参数 |
| 端口 8080 被占用 | 其他程序占用端口 | 修改 `application.yml` 中 `server.port` 或关闭占用程序 |

## 12. 用户操作指南

### 12.1 浏览电影

1. 打开系统后，首页自动加载 TMDB 热门电影列表
2. 电影以 **6 列网格** 形式展示，每张卡片包含海报、标题和评分
3. 向下滚动页面浏览更多电影
4. 点击任意电影卡片，弹出**详情弹窗**，包含：
   - 电影海报（大幅）
   - 标题、上映年份、片长
   - 用户评分（五星进度条）
   - 剧情简介
   - 类型标签

### 12.2 搜索电影

1. 在顶部导航栏的**搜索框**中输入电影名称
2. 输入 3 个字符后，系统自动显示**搜索建议下拉菜单**
3. 点击建议项，或输入完整关键词后按 **Enter** 键
4. 页面切换为搜索结果视图，展示匹配的电影
5. 清空搜索框或点击 Logo 可返回热门电影视图

### 12.3 用户注册与登录

**注册**：

1. 点击导航栏右上角的 **"登录"** 按钮
2. 在弹出的对话框中，点击 **"注册"** 标签页
3. 填写用户名（3-50 字符）、密码（6 位以上）、昵称（可选）
4. 点击 **"注册"** 按钮
5. 注册成功后自动登录

**登录**：

1. 点击导航栏右上角的 **"登录"** 按钮
2. 在 **"登录"** 标签页输入用户名和密码
3. 点击 **"登录"** 按钮
4. 登录成功后：
   - 导航栏右侧显示用户头像和昵称
   - 可访问个人中心页面

**登出**：

点击导航栏用户头像旁的**下拉菜单**，选择 **"退出登录"**。

### 12.4 收藏电影

**添加收藏**：

1. 将鼠标悬停在任意电影卡片上
2. 卡片上浮现操作按钮，点击 **♡（收藏）** 图标
3. 图标变为红色实心，表示收藏成功
4. 如未登录，系统自动弹出登录窗口

**查看收藏**：

1. 点击导航栏用户头像 → **"个人中心"**（或直接访问 `/profile`）
2. 默认显示 **"收藏"** 标签页
3. 所有已收藏电影以列表形式展示（按收藏时间倒序）

**取消收藏**：

在个人中心的收藏列表中，点击电影右侧的**删除按钮**即可取消收藏。

### 12.5 标记想看/看过

**添加标记**：

1. 将鼠标悬停在电影卡片上
2. 点击 **"想看"**（书签图标）或 **"看过"**（对勾图标）
3. 图标高亮表示标记成功

**管理标记**：

1. 进入个人中心（`/profile`）
2. 切换至 **"想看"** 或 **"看过"** 标签页
3. 查看对应列表
4. 可在此切换标记状态或删除标记

> **提示**：同一部电影只能有一个标记状态。已标记为"想看"的电影可以切换为"看过"，反之亦然。

### 12.6 个人中心管理

访问 `/profile` 页面可进行以下操作：

- **编辑昵称**：点击昵称旁的编辑图标，输入新昵称后保存
- **上传头像**：点击头像区域，选择本地图片文件上传（转为 base64 存储）
- **查看收藏**：切换至"收藏"标签页查看已收藏电影
- **查看标记**：切换至"想看"/"看过"标签页查看观影标记

### 12.7 主题切换

- 点击导航栏**右上角的主题切换按钮**（月亮/太阳图标）
- 系统一键切换**深色模式**与**浅色模式**
- 用户偏好保存至浏览器 localStorage，下次访问自动应用

## 13. 系统配置说明

### 13.1 后端配置

配置文件位置：`movie-backend/src/main/resources/application.yml`

```yaml
server:
  port: 8080                          # 服务端口号

spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/movie_recommendation?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root                     # 数据库用户名
    password: your_password            # 数据库密码 ← 需修改

mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true # 驼峰命名转换
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl  # SQL 日志输出（生产环境建议关闭）

jwt:
  secret: MovieRecommendationSystemSecretKey2024ForJwtTokenGeneration  # JWT 签名密钥
  expiration: 604800000                # Token 有效期（毫秒），默认 7 天
```

### 13.2 前端配置

**API 地址**：`movie-frontend/src/api/index.js` 中的 `baseURL`

```javascript
const request = axios.create({
  baseURL: 'http://localhost:8080/api',    // 后端 API 地址
  timeout: 10000                            // 请求超时时间（ms）
})
```

**开发服务器端口**：`movie-frontend/vite.config.js` 中的 `server.port`

```javascript
server: {
  port: 5173,           // 前端开发服务器端口
  open: true            // 启动后自动打开浏览器
}
```

**TMDB API 代理**：`movie-frontend/vite.config.js` 中的 `server.proxy`

```javascript
proxy: {
  '/tmdb': {
    target: 'https://api.themoviedb.org',
    changeOrigin: true,
    rewrite: (path) => path.replace(/^\/tmdb/, '')
  }
}
```
